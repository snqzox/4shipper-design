# Components

One Markdown file per design-system component lives here. Generated and maintained by the
**design-system-manager** agent (descriptions written by the **ds-component-describer** subagent).
The component inventory comes from `data/components.json`.

Covers the **42** documented components/sets currently in the **Transportly UI Kit 2.0**.
(Product compositions that live in the 4Shipper Design file — e.g. Waypoint Entry, Address entry,
Carrier Offer — are intentionally not documented here; this folder tracks the UI Kit library only.)
